# Silex 开发者文档编写指南

本目录 (`docs/src/general_docs`) 下的文档是面向**人类开发者**的官方指南。
它们旨在帮助用户学习、理解和使用 Silex 框架构建应用。

## 适用范围
本指南适用于以下目录及文件：
*   所有根目录下的 Markdown 文件（如 `chapter_*.md`）。
*   各 Crate 的用户指引文档（如 `silex_core/`, `silex_dom/`, `silex_html/` 下的 `README.md`）。


## 核心原则 (Core Principles)

### 1. 教学导向 (Tutorial-Driven)
*   **循序渐进**：文档应从简单的概念入手，逐步深入。避免一上来就堆砌复杂的底层概念。
*   **关注“如何做” (How-to)**：优先展示如何解决具体问题（例如：“如何创建一个计数器”、“如何进行网络请求”），而不是单纯罗列 API。
*   **最佳实践**：在给出方案时，明确指出这是 Silex 推荐的做法，并解释原因（例如：性能、可维护性）。

### 2. 人类可读性 (Readability)
*   **语言流畅**：使用自然、通顺的**中文**。
*   **避免术语堆砌**：如果必须使用专业术语（如 Fine-Grained Reactivity），请在第一次出现时进行简要解释或类比。
*   **格式清晰**：充分利用 Markdown 的标题、列表、引用块和代码块高亮，提高视觉扫描效率。

### 3. 代码示例 (Code Examples)
*   **完整性**：示例代码最好是自包含的，或者用户可以直接复制粘贴运行的片段。
*   **注释**：关键代码行应包含简洁的注释，解释“这行代码在做什么”。
*   **对比**：必要时可以展示“推荐做法” vs “错误做法”，帮助用户避坑。

### 4. 结构一致性
*   **简介**：每个文档开头应简述本章节的内容和目标。
*   **小结**：在长文档的结尾进行简短总结。

请遵循以上原则进行文档撰写和维护，确保社区开发者拥有良好的阅读体验。
